1. Factory Method Pattern

Identification of the Place for Factory Method in the System

In a typical online shopping platform, we have different categories of products (such as electronics, clothing, and books). These categories require different types of products to be created, with each type having its own creation process. The problem arises when we need to create products dynamically based on the user's request, without hardcoding the creation process for each product type.

Pattern Description

The Factory Method Pattern defines an interface for creating an object, but allows subclasses to alter the type of objects that will be created. The client interacts only with the Creator interface, which delegates the task of creating the product to the subclass's implementation.

Pattern Theory and Its Place in the System

The Factory Method Pattern is part of the Creational Design Patterns group, which focuses on controlling the object creation mechanism. It provides a way to delegate the responsibility of creating objects to subclass factories without altering the client code. The Factory Method Pattern fits in here by allowing you to handle the product creation process dynamically based on the category requested by the user. The factories are responsible for creating the appropriate product type.

// Abstract Product
interface Product {
    void getDetails();
    void addToCart();
}

// Concrete Products
class Electronics implements Product {
    @Override
    public void getDetails() {
        System.out.println("Electronics: High-end laptop");
    }

    @Override
    public void addToCart() {
        System.out.println("Electronics added to cart.");
    }
}

class Clothing implements Product {
    @Override
    public void getDetails() {
        System.out.println("Clothing: Blue jeans");
    }

    @Override
    public void addToCart() {
        System.out.println("Clothing added to cart.");
    }
}

class Books implements Product {
    @Override
    public void getDetails() {
        System.out.println("Books: Design Patterns by Erich Gamma");
    }

    @Override
    public void addToCart() {
        System.out.println("Books added to cart.");
    }
}

// Abstract Creator
abstract class Creator {
    public abstract Product createProduct();
    public void someOperation() {
        Product product = createProduct();
        product.getDetails();
        product.addToCart();
    }
}

// Concrete Creators
class ElectronicsFactory extends Creator {
    @Override
    public Product createProduct() {
        return new Electronics();
    }
}

class ClothingFactory extends Creator {
    @Override
    public Product createProduct() {
        return new Clothing();
    }
}

class BooksFactory extends Creator {
    @Override
    public Product createProduct() {
        return new Books();
    }
}

// Client Code
public class Main {
    public static void main(String[] args) {
        // Create products using ElectronicsFactory
        Creator electronicsFactory = new ElectronicsFactory();
        System.out.println("Creating Electronics:");
        electronicsFactory.someOperation();

        // Create products using ClothingFactory
        Creator clothingFactory = new ClothingFactory();
        System.out.println("\nCreating Clothing:");
        clothingFactory.someOperation();

        // Create products using BooksFactory
        Creator booksFactory = new BooksFactory();
        System.out.println("\nCreating Books:");
        booksFactory.someOperation();
    }
}

2. Abstract Factory Pattern

Identification of the Place for Abstract Factory in the System

In an online shopping platform, we may offer furniture collections that include matching chairs and sofas. These collections can belong to different styles, such as Victorian or Modern. The challenge is to ensure that a customer who selects a specific style gets all furniture items from the same collection. Managing these dependencies dynamically without tightly coupling the client code to specific product implementations is essential.

Pattern Description

The Abstract Factory Pattern provides an interface for creating families of related or dependent objects without specifying their concrete classes. This ensures that all products created by the factory are compatible and conform to a particular style or theme. The client interacts with the abstract factory interface, which delegates the creation process to concrete factories for specific product families.

Pattern Theory and Its Place in the System

The Abstract Factory Pattern is part of the Creational Design Patterns group. It is useful when the system needs to deal with multiple families of related objects and enforce consistency between them. In the context of an online shopping platform, this pattern ensures that customers selecting a style (e.g., Victorian or Modern) receive compatible furniture items. The Abstract Factory Pattern allows for scalability, as adding new styles involves creating new concrete factories without modifying the client code.

Source Code:


// Abstract Products
interface Chair {
    void sitOn();
}

interface Sofa {
    void lieOn();
}

// Concrete Products
class VictorianChair implements Chair {
    @Override
    public void sitOn() {
        System.out.println("Sitting on a Victorian chair.");
    }
}

class ModernChair implements Chair {
    @Override
    public void sitOn() {
        System.out.println("Sitting on a Modern chair.");
    }
}

class VictorianSofa implements Sofa {
    @Override
    public void lieOn() {
        System.out.println("Lying on a Victorian sofa.");
    }
}

class ModernSofa implements Sofa {
    @Override
    public void lieOn() {
        System.out.println("Lying on a Modern sofa.");
    }
}

// Abstract Factory
interface FurnitureFactory {
    Chair createChair();
    Sofa createSofa();
}

// Concrete Factories
class VictorianFurnitureFactory implements FurnitureFactory {
    @Override
    public Chair createChair() {
        return new VictorianChair();
    }

    @Override
    public Sofa createSofa() {
        return new VictorianSofa();
    }
}

class ModernFurnitureFactory implements FurnitureFactory {
    @Override
    public Chair createChair() {
        return new ModernChair();
    }

    @Override
    public Sofa createSofa() {
        return new ModernSofa();
    }
}

// Client Code
public class Main {
    public static void main(String[] args) {
        // Create a Victorian furniture family
        FurnitureFactory victorianFactory = new VictorianFurnitureFactory();
        Chair victorianChair = victorianFactory.createChair();
        Sofa victorianSofa = victorianFactory.createSofa();

        System.out.println("Victorian Furniture:");
        victorianChair.sitOn();
        victorianSofa.lieOn();

        // Create a Modern furniture family
        FurnitureFactory modernFactory = new ModernFurnitureFactory();
        Chair modernChair = modernFactory.createChair();
        Sofa modernSofa = modernFactory.createSofa();

        System.out.println("\nModern Furniture:");
        modernChair.sitOn();
        modernSofa.lieOn();
    }
}

3. Conclusion

The Factory Method and Abstract Factory patterns are essential creational design patterns that help in managing object creation in a flexible and scalable manner. The Factory Method Pattern is useful for creating objects of a single type, while the Abstract Factory Pattern is ideal for creating families of related objects. Both patterns ensure that the system remains modular and easy to extend without modifying existing client code.
