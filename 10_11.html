Design Pattern Report

1. Identifying the Place for the Structural Design Pattern in the System

The Adapter Pattern can be used alongside the Factory Method and Abstract Factory patterns. For example, if different product categories or furniture styles use different interfaces, the Adapter Pattern can be used to make these interfaces compatible.

2. Describing the Problem That Justifies the Need for the Design Pattern

Problem: Different product categories or furniture styles use different interfaces. These interfaces are incompatible, and the client code has to work with these different interfaces. This leads to complex code and requires changes to the client code when new products or styles are added.

3. Describing the Pattern, Its Theory, and Its Place in the System

The Adapter Pattern allows two incompatible interfaces to work together. When used with the Factory Method and Abstract Factory patterns, it provides a compatible interface for different product categories or furniture styles. This way, the client code interacts with only one interface, and new products or styles can be added without modifying the client code.

5. Writing the Source Code That Implements the Pattern

Below is an example of Java code that implements the Adapter Pattern alongside the Factory Method and Abstract Factory patterns:


// Product Interface
interface Product {
    void getDetails();
    void addToCart();
}

// Concrete Products
class Electronics implements Product {
    @Override
    public void getDetails() {
        System.out.println("Electronics: High-end laptop");
    }

    @Override
    public void addToCart() {
        System.out.println("Electronics added to cart.");
    }
}

class Clothing implements Product {
    @Override
    public void getDetails() {
        System.out.println("Clothing: Blue jeans");
    }

    @Override
    public void addToCart() {
        System.out.println("Clothing added to cart.");
    }
}

class Books implements Product {
    @Override
    public void getDetails() {
        System.out.println("Books: Design Patterns by Erich Gamma");
    }

    @Override
    public void addToCart() {
        System.out.println("Books added to cart.");
    }
}

// Adapter Class
class ProductAdapter {
    private Product product;

    public ProductAdapter(Product product) {
        this.product = product;
    }

    public void getDetails() {
        product.getDetails();
    }

    public void addToCart() {
        product.addToCart();
    }
}

// Client Code
public class Client {
    public static void main(String[] args) {
        Product electronics = new Electronics();
        ProductAdapter electronicsAdapter = new ProductAdapter(electronics);

        electronicsAdapter.getDetails();
        electronicsAdapter.addToCart();

        Product clothing = new Clothing();
        ProductAdapter clothingAdapter = new ProductAdapter(clothing);

        clothingAdapter.getDetails();
        clothingAdapter.addToCart();

        Product books = new Books();
        ProductAdapter booksAdapter = new ProductAdapter(books);

        booksAdapter.getDetails();
        booksAdapter.addToCart();
    }
}

Conclusion

In these tasks, we used the Adapter Pattern alongside the Factory Method and Abstract Factory patterns to provide a compatible interface for different product categories and furniture styles. This allows the client code to interact with only one interface, and new products or styles can be added without modifying the client code. This makes the code more flexible and extensible.
